# 🛠️ 程序使用说明
## 1. 安装依赖库

在运行程序前，需要先安装必要的Python库：

```bash
pip install requests pandas
```

## 2. 程序功能特点
- 多数据源获取：自动尝试东方财富、腾讯财经、新浪财经等多个数据源

- 实时监控：可设置自动刷新间隔（默认60秒）

- 溢价率计算：自动计算并高亮显示高溢价率（>5%）和折价（<-3%）

- CSV导出：一键导出所有数据到CSV文件

- 基金管理：可随时添加新的LOF基金代码

## 3. 主要操作按钮
- 开始监控：启动自动刷新数据

- 停止监控：停止自动刷新

- 手动刷新：立即获取最新数据

- 导出CSV：将当前数据导出为CSV文件

- 添加基金：添加新的LOF基金代码到监控列表

## 4. 数据字段说明
| 字段 | 说明 |
| :--- | :--- |
| 代码 | LOF基金6位代码 |
| 名称 | 基金简称 |
| 实时价 | 二级市场交易价格 |
| 净值 | 基金单位净值 |
| 溢价率 | (实时价-净值)/净值×100% |
| 涨跌幅 | 当日价格涨跌幅度 |
| 成交量(万) | 成交数量（万手） |
| 更新时间 | 数据最后更新时间 |

## 5. 注意事项
1. 数据源限制：由于免费数据源的限制，某些基金的净值数据可能获取不完整

2. 实时性：数据有15-30秒的延迟，适合监控但不适合高频交易

3. 溢价率计算：需要同时获得实时价格和净值，部分基金可能缺失净值数据

4. 错误处理：程序有基本的错误处理，但网络异常时可能获取失败

## 6. 自定义扩展建议
1. 增加更多数据源：可以添加天天基金网、雪球等数据接口

2. 添加报警功能：当溢价率达到设定阈值时弹出提醒

3. 历史数据记录：自动保存历史数据用于分析

4. 优化界面：添加图表展示溢价率变化趋势

# ⚠️ LOF套利操作核心流程（T+2周期）

## 整个过程需要三个交易日，关键操作和时间点如下：

### T日（申购日）

- 关键操作：在交易日15:00前，通过证券APP完成场外申购。

- 操作路径：交易 → 场内基金 → 基金申购 → 输入代码161226 → 输入金额（A类上限500元）。

- 重要提示：申购按当日净值计算，需支付申购费（通常1%-1.5%）。

### T+1日（确认日）

- 关键状态：基金份额在当晚（约22:00后）确认至账户，但不可交易。

- 重要提示：此日无法进行任何卖出操作。

### T+2日（可卖日）

- 关键操作：基金份额可在二级市场像股票一样卖出。

- 卖出时机：通常建议开盘后尽早卖出，以锁定溢价，避免价格波动。

- 重要提示：卖出时按实时交易价格成交，需支付券商交易佣金。

## 必须警惕的三大核心风险
这不是“稳赚”的羊毛，主要风险在于“T+2”的等待期：

1. **溢价收敛风险：**这是最大风险。如果2天后溢价率大幅降低甚至消失，你的卖出价可能无法覆盖申购成本和交易费用，导致直接亏损。历史上类似套利曾因溢价快速收敛导致投资者亏损。

2. **流动性风险：**该基金场内流通盘较小。如果T+2日当天有大量套利盘集中卖出，可能出现无法及时卖出或只能“折价”卖出的情况。

3. **白银价格波动风险：**基金净值跟踪白银期货价格。在等待的2天里，如果白银价格大幅下跌，即使溢价不变，基金净值也会缩水，侵蚀你的套利空间。

## 重要注意事项
- 选择正确份额：如果你想通过“申购-卖出”套利，必须申购A类份额，因为只有A类份额可以转入场内卖出。C类份额主要用于场外持有。

- 收益计算示例：以申购500元A类份额、溢价率50%理想情况估算，潜在收益约250元，但需扣除申购费（约5-7.5元）和卖出佣金。实际收益取决于卖出时的溢价率，且基金公司已多次提示当前高溢价不可持续。

- 关注官方公告：该基金近期频繁发布溢价风险提示，并可能临时停牌，这会影响你的卖出计划，操作前务必查看最新公告。
